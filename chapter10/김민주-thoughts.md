# 느낀점

이 책을 읽으면서 맵리듀스라는 단어를 정말 많이 봤다.
앞선 챕터들에서도 분산 처리 얘기가 나올 때마다 맵리듀스가 언급됐고,
Hadoop, HDFS 같은 단어들도 자주 나왔다.

그런데 솔직히 말하면 "데이터를 나눠서 처리하는 무언가" 정도로만 알고 있었다.
개념을 아는 척했던 거다.


단점을 보니 왜 느린지 이해됐다.

```
MR1 → HDFS 저장 → MR2 → HDFS 저장 → MR3 → HDFS 저장 → 결과
```

여러 단계를 체인으로 연결하면 중간 결과를 매번 HDFS에 쓴다.
장애 복구를 위한 설계였지만 그 대가로 속도를 희생한 것이다.

트레이드오프를 당연하게 받아들이는 시각이 이 책을 읽으면서 계속 길러지는 것 같다.
맵리듀스가 나쁜 게 아니라, 당시의 제약 조건 안에서 최선의 선택이었던 것이다.

Spark가 왜 맵리듀스보다 빠른지 이해됐다.

단순히 "메모리에서 처리해서 빠르다"는 말을 들었을 때는 그냥 그런가 보다 했는데
맵리듀스가 왜 디스크에 저장하는지 그 비용이 얼마나 큰지를 이해하고 나니
Spark의 접근 방식이 왜 혁신적이었는지 납득이 됐다.

맵리듀스 → Spark로 이어지는 흐름을 보면서,
모든 새로운 기술은 이전 기술의 **불편함을 참다 못해** 나온 것이라는 걸 다시 느꼈다.

Spark를 처음 배울 때 "그냥 Hadoop보다 빠른 것"으로만 알고 있었는데,
이제는 Spark가 무엇을 해결하려 했는지, 어떤 트레이드오프를 선택했는지가 보인다.
이 차이가 꽤 크다고 생각한다.

Chapter 11은 스트림 처리다.
일괄 처리가 "유한한 데이터"를 다뤘다면,
스트림 처리는 끝없이 흘러오는 데이터를 어떻게 다루는가의 이야기다.

맵리듀스를 이해한 지금 스트림 처리가 일괄 처리의 어떤 한계를 어떻게 극복하는지
더 명확하게 보일 것 같아서 기대가 된다.
