## 최종 핵심 추리기

1.  데이터 인코딩이란 이 책에서는 Serialize를 의미함!!
2.  언어 특화 Serialization은 현대 아키텍처에서는 지양해야함. 언어 끼리 호환이 안되어서 MSA나 Service-based에서 서로 통신이 어려움!
3.  JSON은 스타트업 디폴트로 거의 99%의 경우 좋으니 gRPC 겉멋 X!
4.  gRPC가 빛을 발하는 경우는:
    - 사내 네트워크 비용이 개발자 gRPC 개발자 월급 상회
    - 백/프론트/MSA 통신시 스키마가 자주 바뀌고 그걸 소통 없이 빠르게 서로 호환되게 하기 위해
    - 모바일 게임 같이 극도로 성능 튜닝이 필요할 때
5.  DB나 통신 스키마나 상위 호환성 하위 호환성 다 지켜야함:
    - 상위 호환성: 모르는 것에 대해 null처리가 대표적
    - 하위 호환성: 예전 데이터를 적절히 처리!
